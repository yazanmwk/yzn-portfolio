## What is this?

**YZN-OS** is a playful, interactive front-end: a full-screen “desktop” with wallpaper, icons, a Start menu, and classic window chrome. Your work and story live inside familiar OS metaphors—folders, terminals, and dialog boxes—so visitors explore instead of scroll a long single page.

## Feature highlights

### Boot into the experience

The first thing visitors see is not a hero section—it’s a **kernel-style boot log** and bold **YZN OS** branding. It sets the tone: this site is an OS simulation, not a generic template.

---

### Desktop, wallpaper, and welcome flow

<p align="center">
  <img src="<img src="https://yazanmwk.vercel.appdocs/readme/desktop-welcome.png" alt="YZN-OS desktop with Welcome window" width="920">
</p>

- **Live desktop** — Wallpaper, desktop icons, and a persistent **taskbar** with clock and window buttons.
- **Welcome window** — Onboarding with a table-of-contents sidebar (Explore, Contact, Have Fun, Thank You) and a clear path into the desktop.
- **Start menu** — Launch apps the way you would on a classic PC.
- **Themes & display** — Tune wallpaper and appearance (including a Display Properties–style experience in Settings).

---

### Apps, games, and utilities

<p align="center">
  <img src="<img src="https://yazanmwk.vercel.appdocs/readme/desktop-apps.png" alt="YZN-OS with Terminal and Paint open" width="920">
</p>

- **YZN Terminal** — Shell-style interface with commands, boot messaging, and room for Easter eggs.
- **MS Paint–style canvas** — Drawing tools, palette, and classic window framing.
- **Built-in “programs”** — Info, Projects, Contact, Settings, Algorithm visualizer, media/image tooling, and more—each in its own window.
- **Retro games** — Treats like **DOOM**, **Quake**, **Solitaire**, **Minesweeper** (desktop-first where noted; mobile gets sensible messaging).
- **Window manager** — Open, focus, minimize, and stack windows like a real desktop.

## Why it exists

Most portfolios optimize for skimming. **YZN-OS** optimizes for **curiosity**: people who want to click icons, read fake “system” copy, and feel like they discovered your work instead of being handed a PDF.

---

## Public GitHub vs the live site (Vercel)

- **Vercel** hosts the real Next.js app. Edits to this README or to `docs/` do **not** change how Vercel builds or serves the site.
- **GitHub** always shows a file browser above the README on a **public** repository. There is no supported way to make “README only” for anonymous visitors while keeping the same public repo and full source in that repo.
- **If you need strangers to see documentation only:** use a **private** repository for the code (Vercel can still deploy from it with the same Git integration), or keep a **small public repo** that only contains this README + images and links to your production URL, with the full codebase elsewhere.

Set **GitHub → Repository → Settings → General → Website** to [https://yazanmwk.vercel.app/](https://yazanmwk.vercel.app/) so the “About” panel points people to the app first.
